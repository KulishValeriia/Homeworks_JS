'number' + 3 + 3: результат number33, тому що 'number' - це строка, JS не може перевести її до числа і робить конкатенацію (обєднання рядків).

null + 3: результат 3, оскільки JS null перетворює на 0, 0+3=3.

5 && "qwerty": результат qwerty. В цій операції JS перевіряє значення зліва направо - що вони true або false, і виводить останнє true значення. 
У цьому випадку обидва значення - true: число 5 - істинне, як таке що відмінне від 0 та NaN; строка "qwerty" також є істинною, 
оскільки будь-яка непуста строка вважається істинною.

+'40' + +'2' + "hillel": результат 42hillel. Унарний плюс (+) перед рядком наводить її до числа: +'40' перетвориться на число 40, а +'2' перетвориться на число 2.
Операція між числами виконує арифметичне додавання: 40 + 2 дасть 42. І "+" між числом та рядком виконує конкатенацію рядка.

'10' - 5 === 6: результат false. У лівій частині порівняння строка '10' перетвориться на число і результат операції =5; оператор строгої рівності (===) - 
порівнює два значення на рівність як за значенням, так і за типом даних. Повертає true, якщо значення рівні і мають однаковий тип, і false - якщо вони різні. 5 не 
дорівнює 6, хоч вони обидва однакові за типом даних.  

true + false: результат 1, оскільки в арифметичних операціях true=1, false=0.

'4px' - 3: результат NaN, оскільки '4px' - не числовий рядок, а спроба виконати арифметичну операцію з нечисловим рядком призводить до NaN (тільки рядок+число
приводить до обєднання).

'4' - 3: результат 1, тому що в цьому випадку рядок '4' перетворюється в number і виконується звичайна арифметична операція.

'6' + 3 ** 0: результат 61, тут "+" виконує конкатенацію між рядком '6' та числом 3 в ступені 0 = 1.

12 / '6': результат 2, тут виконана арифметична операція 12/6 - '6' приведено до числа.

'10' + (5 === 6): результат 10false. (5 === 6) - порівняння невірне (false), "+" виконує конкатенацію між рядком '10' і false.

null == '': результат false. == - порівнює два значення на рівність за значенням: null не приводиться до інших примітивних типів, крім undefined,
а порожній рядок '' має значення falsе, але при порівнянні він приведений до 0. Тобто null не рівний 0.

3 ** (9 / 3): результат 27. Спочатку виконана дія в дужках 9/3=3, а потім 3 зведено у ступінь 3.

!!'false' == !!'true': результат true. == - порівнює два значення на рівність за значенням: 'false' і 'true' мають однакове значення - не пустий рядок.

0 || '0' && 1: результат 1. Порівняння відбувається зліва направо: 1. || - або 0 або '0' - обидва значення є true, 2. && - і true (=1) і 1 також є істинними.
(&& повертає останнє значення якщо всі операнди true).

(+null == false) < 1: результат false. +null(це-false) == false - true, true < 1 - false тому що true=1, а 1 не може бути < 1 (1=1).

false && true || true: результат true. false && true - && (і) повертає перший операнд, який інтерпретований як false, false || true - || повертає true,
якщо хоча б один із аргументів true.

false && (false || true): результат false. 1 - (false || true) - || повертає true, якщо хоча б один із аргументів true. 2 - false && true -
повертає перший операнд, який інтерпретований як false.

(+null == false) < 1 ** 5: результат false. 1 - +null(це-false) == false є true, 2 - 1 ** 5 = 1, 3 - true < 1 - false тому що true=1, а 1 не може бути < 1 (1=1).
